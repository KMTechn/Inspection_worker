  ┣━ 설명: 전체 시스템 자동 테스트 실행

  ┃   ┣━ 트레이 완료 및 라벨 생성 테스트

  ┃   ┣━ 잔재 생성 세션 테스트

      ┣━ 팰릿 수량: 기본 1개
      ┣━ 리워크 수량: 기본 1개
      ┣━ 잔재 수량: 기본 3개
      ┗━ 불량품 통합 수량: 기본 5개

🔧 2. 불량 모드 테스트 시스템

_TEST_DEFECT_MODE_
  ┣━ 설명: **완전한 불량시스템 통합 테스트** 🔥 (UPDATED!)
  ┣━ 특징: 품목 선택 → 불량품 생성 → 불량품 통합 → 불량표 생성
  ┣━ **NEW! 품목 선택 UI**: _RUN_AUTO_TEST_와 동일한 품목 선택 다이얼로그
  ┣━ 포함 기능:
  ┃   ┣━ 🎯 품목 선택 다이얼로그 (콤보박스에서 선택)
  ┃   ┣━ 🔍 표준 검사 모드에서 F12+바코드로 실제 불량품 생성
  ┃   ┣━ 📊 실제 로그 파일에 불량품 데이터 자동 등록
  ┃   ┣━ 🔄 실제 불량품 바코드로 통합 작업
  ┃   ┣━ 📄 불량표 실제 생성 및 파일 저장
  ┃   ┗━ 📋 잔량표 생성 테스트 (선택사항)
  ┣━ **실제 제품과 동일한 워크플로우**:
  ┃   ┣━ **0단계**: 품목 선택 다이얼로그에서 테스트할 품목 선택
  ┃   ┣━ **1단계**: 선택된 품목으로 현품표 스캔 (목표수량: 10개)
  ┃   ┣━ **2단계**: 양품 10개 + 중간에 불량품 5개 혼합 스캔 (F12)
  ┃   ┣━ **3단계**: 양품 10개 달성으로 작업 자동 완료
  ┃   ┣━ **4단계**: 불량품 통합 모드로 자동 전환
  ┃   ┣━ **5단계**: 실제 로그에서 불량품 데이터 자동 로드
  ┃   ┣━ **6단계**: 실제 불량품 바코드로 통합 작업 (5개)
  ┃   ┣━ **7단계**: 불량표 실제 생성 (PNG + JSON)
  ┃   ┗━ **8단계**: 잔량표 생성 테스트 (선택사항)
  ┣━ **주요 개선사항**:
  ┃   ┣━ ✅ 현품표 수량: QT=10 (불량품은 목표수량에 미포함)
  ┃   ┣━ ✅ 현실적 스캔: 양품 10개 완료 + 중간 불량품 5개
  ┃   ┣━ ✅ 실제 동작 검증: 테스트와 실제가 동일한 로그 경로
  ┃   ┗━ ✅ 불량 모드에서 현품표 없이도 바코드 입력 가능
  ┗━ 생성 파일:
      ┣━ 검사 로그: C:/Sync/검사작업이벤트로그_*.csv
      ┣━ 불량표: C:/Sync/labels/defective_labels/YYYY-MM-DD/불량표_DEFECT-ID.png
      ┣━ 불량 데이터: C:/Sync/defects_data/YYYY-MM-DD/DEFECT-ID.json
      ┣━ 잔량표: C:/Sync/labels/remnant_labels/YYYY-MM-DD/잔량표_SPARE-ID.png
      ┗━ 잔량 데이터: C:/Sync/remnants/YYYY-MM-DD/SPARE-ID.json

_TEST_F12_DEFECT_
  ┣━ 설명: **F12 페달 불량 검출 로직 테스트** (표준 검사 모드)
  ┣━ 시뮬레이션 내용:
  ┃   ┣━ 현품표 QR 코드 스캔으로 작업 시작
  ┃   ┣━ 양품 바코드 스캔 (일반 스캔)
  ┃   ┣━ 불량품 바코드 스캔 (F12 페달 시뮬레이션 + 바코드)
  ┃   ┣━ 양품/불량품 실시간 분류 검증
  ┃   ┗━ 목표 수량 달성 시 자동 완료
  ┗━ 테스트 시나리오:
      ┣━ 양품 7개, 불량품 3개 스캔
      ┣━ F12 페달 시뮬레이션으로 불량품 분류
      ┣━ 실시간 UI 피드백 확인
      ┗━ 완료된 세션의 양품/불량품 데이터 검증

_TEST_DEFECT_MERGE_
  ┣━ 설명: 불량품 통합 모드만 단순 테스트
  ┣━ 시뮬레이션 내용:
  ┃   ┣━ 불량품 통합 세션 시작
  ┃   ┣━ 목표 수량 설정 (기본 48개 → 테스트용 10개)
  ┃   ┣━ 불량품 바코드 10개만 스캔
  ┃   ┗━ 진행률 확인만 (불량표 생성 안함)
  ┗━ 용도: 불량품 통합 UI 및 진행률 확인용

_TEST_DEFECT_QR_SCAN_  (NEW! 🎯)
  ┣━ 설명: **불량표 QR 스캔 기능 통합 테스트**
  ┣━ 특징: 불량표 QR 코드 스캔 → 초과분 처리 → 자동 분할
  ┣━ 테스트 시나리오:
  ┃   ┣━ 🔧 불량품 통합 모드로 자동 전환
  ┃   ┣━ 📋 첫 번째 불량표 QR 스캔 (25개 불량품)
  ┃   ┣━ 🎯 목표 수량 미달성으로 정상 추가
  ┃   ┣━ 📋 두 번째 불량표 QR 스캔 (30개 불량품)
  ┃   ┣━ ⚠️ 총 55개 > 목표 48개 → 초과분 7개 감지
  ┃   ┣━ 🤖 테스트 모드에서 자동 분할 처리
  ┃   ┣━ 📊 메인: 48개로 불량표 생성
  ┃   ┗━ 📄 초과분: 7개로 새 불량표 자동 생성
  ┣━ **핵심 기능 검증**:
  ┃   ┣━ 불량표 JSON 데이터 파일 자동 생성
  ┃   ┣━ 불량표 QR 코드 인식 및 파싱
  ┃   ┣━ 초과 수량 감지 및 자동 분할
  ┃   ┣━ 초과분 새 불량표 자동 생성
  ┃   ┗━ 불량품 바코드 목록 로딩 및 추가
  ┗━ 용도: 불량표 QR 스캔 워크플로우 전체 검증

   _TEST_UI_RESPONSIVE_ → _TEST_SOUND_SYSTEM_ → 사용자 경험 확인

🔧 **불량 처리 모드 개선사항**:
• **현품표 없이 바코드 스캔 가능**:
  - 불량 처리 모드에서 바코드만으로 작업 시작
  - 바코드의 품목코드를 자동 감지하여 세션 시작
  - 기본 목표수량 48개로 자동 설정

• **자동 불량품 기록**:
  - available_defects에 없는 바코드도 직접 INSPECTION_DEFECTIVE 이벤트 생성
  - direct_scan 플래그로 현품표 없이 스캔된 항목 구분
  - 품목코드 기반 검증으로 잘못된 바코드 방지

• **통합된 로그 시스템**:
  - 테스트와 실제 작업이 동일한 로그 파일에 저장
  - C:/Sync/검사작업이벤트로그_*.csv 형태로 통일
  - 실제 제품 동작과 완전히 동일한 방식

🎯 **_TEST_DEFECT_MODE_ 개선사항**:
• **품목 선택 UI 추가**: _RUN_AUTO_TEST_와 동일한 선택 다이얼로그
• **현실적 수량 관리**: 현품표 QT=10, 불량품은 목표수량에 미포함
• **혼합 스캔 로직**: 양품 10개 달성 + 중간에 불량품 5개 삽입
• **실제 동작 검증**: 테스트가 실제 제품과 동일한 방식으로 작동
