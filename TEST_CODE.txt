🔧 수동 테스트 시나리오 가이드
===============================================================================

🚀 **기본 테스트 흐름**:
1. 로그인: 작업자 이름 입력 → "작업 시작" 클릭
2. 모드 확인: 표준 검사 모드인지 확인
3. 현품표 스캔: 위의 샘플 현품표 중 하나 선택하여 스캔
4. 제품 바코드 스캔: 해당 품목의 제품 바코드들을 순서대로 스캔
5. 불량품 테스트: F12 키를 누른 상태로 바코드 스캔 (불량품 처리)
6. 완료 확인: 목표 수량 달성 시 자동 완료 및 현품표 생성

**시나리오 1: 기본 검사 테스트 (생산 형식)** (10개 목표)
1. 현품표 스캔: PHS=1|CLC=AAA2270730100|WID=MFG-WO-2025-00047|SPC=L07|FPB=L07000306|OBD=2025-09-25|PJT=KMC_LHD|QT=10
2. 양품 스캔 (7개): PROD-KMC-LHD-AAA2270730100-20250925001 ~ 007
3. 불량품 스캔 (F12 + 3개): PROD-KMC-LHD-AAA2270730100-20250925008 ~ 010
4. 자동 완료 및 현품표 생성 확인

**시나리오 2: 대량 검사 테스트 (생산 형식)** (18개 목표)
1. 현품표 스캔: PHS=1|CLC=AAA2287560100|WID=MFG-WO-2025-00050|SPC=Q06|FPB=Q06000309|OBD=2025-09-25|PJT=KMC_RHD|QT=18
2. 양품 12개 → 불량품 3개 (F12) → 양품 3개 (총 15개 양품, 3개 불량품)
3. 제품 바코드: PROD-KMC-RHD-AAA2287560100-20250925001 ~ 018 사용
4. 실시간 통계 및 진행률 확인

**시나리오 3: 최대 수량 테스트 (생산 형식)** (20개 목표)
1. 현품표 스캔: PHS=1|CLC=AAA2270680000|WID=MFG-WO-2025-00052|SPC=M04|FPB=M04000311|OBD=2025-09-25|PJT=HMC_LHD|QT=20
2. 모든 제품 바코드 사용: PROD-HMC-LHD-AAA2270680000-20250925001 ~ 020
3. 양품/불량품 혼합 스캔 (F12 키 활용)
4. 세션 완료 및 라벨 생성 확인

**시나리오 1-B: 기본 검사 테스트 (구형 생산 형식)** (10개 목표)
1. 현품표 스캔: CLC=AAA2270730100|WID=WO-2025-L07-001|QT=10
2. 양품 스캔 (7개): PROD-KMC-LHD-AAA2270730100-20250925001 ~ 007
3. 불량품 스캔 (F12 + 3개): PROD-KMC-LHD-AAA2270730100-20250925008 ~ 010
4. 자동 완료 및 현품표 생성 확인

**시나리오 1-C: 기본 검사 테스트 (테스트 형식)** (10개 목표)
1. 현품표 스캔: CLC=INSPECTION|WID=TEST-KMC-LHD-001|ITEM=AAA2270730100|QTY=10|DATE=20250925
2. 양품 스캔 (7개): PROD-KMC-LHD-AAA2270730100-20250925001 ~ 007
3. 불량품 스캔 (F12 + 3개): PROD-KMC-LHD-AAA2270730100-20250925008 ~ 010
4. 자동 완료 및 현품표 생성 확인

**시나리오 4: 불량품 통합 모드 테스트**
1. 불량품 통합 모드로 전환 ("불량품 통합 모드로" 버튼 클릭)
2. 이전 불량품이나 직접 바코드 스캔 (예: PROD-HMC-RHD-AAA2287530000-20250925001 ~ 020)
3. 48개까지 스캔하여 자동 불량표 생성 확인

**시나리오 5: 연속 세션 테스트**
1. 첫 번째 세션: TEST-KMC-LHD-001 (10개) → 완료
2. 두 번째 세션: TEST-KMC-LHD-002 (15개) → 완료
3. 동일 품목 연속 처리 및 통계 누적 확인

**시나리오 6: 리워크 모드 테스트**
1. 기본 검사에서 불량품 생성 (F12 + 바코드)
2. 리워크 모드로 전환 ("리워크 모드로" 버튼 클릭)
3. 불량품 바코드를 다시 스캔하여 양품으로 복구
4. 리워크 통계 확인

**시나리오 7: 잔량 모드 테스트**
1. 잔량 모드로 전환 ("잔량 생성 모드로" 버튼 클릭)
2. 잔여 제품 바코드들을 스캔
3. "잔량표 생성" 버튼으로 잔량표 생성
4. 생성된 잔량표를 새로운 검사 세션에서 활용

📝 주의사항
===============================================================================

• **테스트 환경 준비**:
  - C:\Sync\ 폴더가 존재하는지 확인
  - assets/Item.csv 파일에 테스트 품목이 등록되어 있는지 확인
  - 바코드 스캐너 또는 키보드 입력 준비

• **테스트 데이터 관리**:
  - 테스트 후 생성된 파일들을 정기적으로 정리
  - 실제 작업 데이터와 테스트 데이터 구분 관리

• **F12 불량품 테스트**:
  - F12 키를 누른 상태에서 바코드 스캔해야 불량품으로 처리
  - 불량품 표시등(빨간색) 확인 후 스캔

• **모드별 테스트**:
  - 각 모드(검사/리워크/잔량/불량통합)는 독립적으로 동작
  - 모드 전환 시 이전 세션 데이터는 자동 저장

• **현품표 바코드 형식** (모두 지원):
  - 신형 생산 현품표: PHS=x|CLC=xxx|WID=xxx|SPC=xxx|FPB=xxx|OBD=xxx|PJT=xxx|QT=xxx
  - 구형 생산 현품표: CLC=xxx|WID=xxx|QT=xxx (최소 필수 필드)
  - 테스트 현품표: CLC=INSPECTION|WID=xxx|ITEM=xxx|QTY=xxx|DATE=xxx
  - JSON 형식: {"CLC": "xxx", "WID": "xxx", "QT": xxx, ...}

• **제품 바코드 형식**:
  - 제품바코드: 13자 이상, 품목코드 포함 권장
  - 실제 바코드 스캐너 사용 시 더욱 정확한 테스트 가능
